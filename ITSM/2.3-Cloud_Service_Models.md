## **2. Platform as a Service (PaaS)**

ðŸ‘‰ **What it is:**

ðŸ‘‰ **Example providers:** AWS Elastic Beanstalk, Google App Engine, Microsoft Azure App Service.

ðŸ‘‰ **Use case:** Building apps quickly without worrying about servers or infrastructure management.

ðŸ‘‰ **Responsibilities:**

## **3. Software as a Service (SaaS)**

ðŸ‘‰ **What it is:**

ðŸ‘‰ **Example providers:** Gmail, Microsoft 365, Dropbox, Salesforce.

ðŸ‘‰ **Use case:** End-users who need fully functional software without setup or management.

## **4. Function as a Service (FaaS) / Serverless**

ðŸ‘‰ **What it is:**

ðŸ‘‰ **Example providers:** AWS Lambda, Google Cloud Functions, Azure Functions.

## **5. Database as a Service (DBaaS)** *(sometimes grouped under PaaS)*

## **Comparison Summary**

| Feature              | IaaS                  | PaaS               | SaaS              | FaaS                |
| -------------------- | --------------------- | ------------------ | ----------------- | ------------------- |
| **User manages**     | OS, apps, data        | Apps, data         | Only data usage   | Just code functions |
| **Provider manages** | Hardware, networking  | Hardware + runtime | Everything        | Infra + runtime     |
| **Best for**         | Control & flexibility | Fast development   | Ready-to-use apps | Event-driven apps   |
| **Example**          | AWS EC2               | Google App Engine  | Gmail             | AWS Lambda          |
