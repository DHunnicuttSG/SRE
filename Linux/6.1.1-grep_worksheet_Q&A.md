# Grep Practice Worksheet

This worksheet uses the `grep_practice` dataset you downloaded. 

---
## ‚úÖ Easy Level

### 1. Find lines containing `error` in `log.txt`
**Command:**
```bash
grep "error" log.txt
```
**Answer:**
Matches lines with lowercase `error` only (none in `log.txt`, because errors are uppercase `ERROR`).

### 2. Case-insensitive search for `warning` in `system.log`
**Command:**
```bash
grep -i "warning" system.log
```
**Answer:**
```
warning: deprecated kernel parameter in cmdline
WARNING: fan speed controller unavailable
```

### 3. Count occurrences of `success` in `output.txt`
**Command:**
```bash
grep -c "success" output.txt
```
**Answer:**
```
3
```
(Counts lowercase `success` only.)

### 4. Show line numbers for `failed` in `report.txt`
**Command:**
```bash
grep -n "failed" report.txt
```
**Answer:**
```
2:Line two: failed due to timeout
5:Line five: failed network check
```

## üîç Intermediate Level

### 5. Search recursively for `TODO` in Python files under `projects`
**Command:**
```bash
grep -r "TODO" projects --include="*.py"
```
**Answer:**
```
projects/app/main.py:# TODO: refactor initialization
projects/lib/utils.py:# TODO: write unit tests for parse_config
```

### 6. Match whole word `cat` in `animals.txt`
**Command:**
```bash
grep -w "cat" animals.txt
```
**Answer:**
```
The cat sat on the mat.
A bobcat is a wild cat.
```

### 7. Invert match to exclude `DEBUG` in `app.log`
**Command:**
```bash
grep -v "DEBUG" app.log
```
**Answer:**
```
INFO Initialized subsystems
INFO User login alice
INFO User login bob
INFO Session created
```

### 8. Regex: Lines starting with `ERROR` in `server.log`
**Command:**
```bash
grep "^ERROR" server.log
```
**Answer:**
```
ERROR Failed to start server
ERROR at startup: missing certificate
```

## üí™ Advanced Level

### 9. Multiple patterns: `error` or `fail` in `system.log`
**Command:**
```bash
grep -E "error|fail" system.log
```
**Answer:**
```
Error: failed to mount /dev/sda1
error: fsck required on /dev/sda1
FAIL: service network.target
```

### 10. Extract IPv4 addresses from `access.log`
**Command:**
```bash
grep -Eo '([0-9]{1,3}\.){3}[0-9]{1,3}' access.log
```
**Answer:**
```
192.168.0.1
10.0.0.254
203.0.113.5
255.255.255.255
8.8.8.8
```

### 11. Show 3 lines of context around `critical` in `kernel.log`
**Command:**
```bash
grep -C 3 -i "critical" kernel.log
```
**Answer:**
```
[    0.000000] Booting Linux Kernel 6.9.0
[    1.000000] critical: thermal zone overheat
[    1.500000] info: enabling swap
[    2.000000] CRITICAL: GPU voltage drop
[    3.000000] warning: clock skew detected
```

### 12. List filenames with `port=8080` under `etc`
**Command:**
```bash
grep -rl "port=8080" etc
```
**Answer:**
```
etc/app/config.conf
```

### 13. Find dates in `YYYY-MM-DD` format in `data.txt`
**Command:**
```bash
grep -E '[0-9]{4}-[0-9]{2}-[0-9]{2}' data.txt
```
**Answer:**
```
Event A on 2025-01-31 was successful
Another date: 1999-12-01
```
