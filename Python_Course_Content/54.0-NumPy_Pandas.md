# Lesson 1: NumPy — Numerical Computing with Arrays

## What NumPy Is (and Why It Exists)

NumPy is the foundation of scientific and data computing in Python. It provides:

*   A fast, memory‑efficient **array object**
*   Vectorized operations that replace slow Python loops
*   A rich set of mathematical and statistical functions

Most data science libraries (including Pandas, SciPy, scikit‑learn) are built on NumPy.

***

## 1. NumPy Arrays and Data Types

### Concept

A **NumPy array** (`ndarray`) is:

*   A fixed-size, homogeneous (single data type) collection of values
*   Stored contiguously in memory (much faster than Python lists)
*   Designed for numeric computation

### Creating Arrays

```python
import numpy as np

# From a Python list
arr = np.array([1, 2, 3, 4])
print(arr)
print(type(arr))
```

```python
# Common array constructors
zeros = np.zeros(5)           # [0, 0, 0, 0, 0]
ones = np.ones((2, 3))        # 2x3 array of ones
range_arr = np.arange(0, 10)  # [0, 1, 2, ..., 9]
linspace = np.linspace(0, 1, 5)  # 5 evenly spaced values

print(zeros)
print(ones)
print(linspace)
```

### Data Types (`dtype`)

NumPy enforces a single data type per array.

```python
arr = np.array([1, 2, 3])
print(arr.dtype)

float_arr = np.array([1.0, 2.0, 3.0])
print(float_arr.dtype)
```

You can specify a dtype explicitly:

```python
arr = np.array([1, 2, 3], dtype=np.float64)
```

**Common Pitfall**

*   NumPy will **upcast automatically**:
    ```python
    np.array([1, 2.5, 3])  # all become floats
    ```

***

## 2. Indexing, Slicing, and Reshaping

### Indexing and Slicing (1D)

```python
arr = np.array([10, 20, 30, 40, 50])

print(arr[0])     # 10
print(arr[-1])    # 50
print(arr[1:4])   # [20, 30, 40]
```

### Indexing (2D)

```python
matrix = np.array([
    [1, 2, 3],
    [4, 5, 6]
])

print(matrix[0, 1])    # Row 0, Column 1 → 2
print(matrix[:, 0])    # First column
print(matrix[1, :])    # Second row
```

### Reshaping Arrays

Reshaping does **not copy data** (when possible).

```python
arr = np.arange(6)
reshaped = arr.reshape((2, 3))

print(reshaped)
```

```text
[[0 1 2]
 [3 4 5]]
```

**Common Pitfall**

*   Total elements must match:
    ```python
    arr.reshape((4, 2))  # ❌ ValueError
    ```

***

## 3. Broadcasting and Vectorized Operations

### Vectorization

Instead of loops:

```python
numbers = np.array([1, 2, 3, 4])
squared = numbers ** 2
print(squared)
```

This is:

*   Cleaner
*   Faster
*   More expressive

### Broadcasting (Mental Model)

Broadcasting allows NumPy to apply operations between arrays of **different shapes**.

```python
arr = np.array([1, 2, 3])
print(arr + 10)
```

```text
[11 12 13]
```

With 2D arrays:

```python
matrix = np.array([[1, 2, 3],
                   [4, 5, 6]])

print(matrix + np.array([10, 20, 30]))
```

**Key Rule**

*   Dimensions must be compatible (align from the end)

**Common Pitfall**

*   Shape mismatch errors:
    ```python
    matrix + np.array([1, 2])  # ❌ incompatible shapes
    ```

***

## 4. Mathematical and Statistical Functions

### Elementwise Math

```python
arr = np.array([1, 4, 9])

print(np.sqrt(arr))
print(np.log(arr))
print(np.exp(arr))
```

### Aggregations

```python
data = np.array([10, 20, 30, 40])

print(data.sum())
print(data.mean())
print(data.min())
print(data.max())
print(data.std())
```

### Along Axes (2D)

```python
matrix = np.array([
    [1, 2, 3],
    [4, 5, 6]
])

print(matrix.sum(axis=0))  # Column-wise
print(matrix.sum(axis=1))  # Row-wise
```

***

## 5. Practical Use Cases and Best Practices

### Typical Use Cases

*   Numerical simulations
*   Feature matrices in ML
*   Efficient data transformations
*   Image and signal processing

### Best Practices

✅ Use vectorization instead of loops  
✅ Use `.shape` to reason about arrays  
✅ Prefer NumPy arrays over lists for math  
✅ Learn slicing well—it unlocks power

# Lesson 2: Pandas — Working with Structured Data

## What Pandas Is (and Why It Exists)

Pandas builds on NumPy and adds:

*   Labeled data structures
*   Spreadsheet‑like operations
*   Powerful data cleaning and aggregation

Think:

*   NumPy → arrays
*   Pandas → **tables with meaning**

***

## 1. Series and DataFrames

### Series (1D labeled data)

```python
import pandas as pd

s = pd.Series([10, 20, 30], index=["a", "b", "c"])
print(s)
```

Access by label or position:

```python
print(s["a"])
print(s.iloc[0])
```

***

### DataFrames (2D tabular data)

```python
data = {
    "name": ["Alice", "Bob", "Charlie"],
    "age": [25, 30, 35],
    "salary": [70000, 80000, 90000]
}

df = pd.DataFrame(data)
print(df)
```

***

## 2. Loading and Inspecting Data

### Loading Data (CSV)

```python
df = pd.read_csv("employees.csv")
```

### Inspecting Structure

```python
df.head()       # First 5 rows
df.tail()       # Last 5 rows
df.info()       # Columns, types, missing values
df.describe()   # Summary statistics
```

**Common Pitfall**

*   Forgetting that `df.head()` returns a DataFrame; it doesn’t print unless shown.

***

## 3. Indexing, Selecting, and Filtering

### Column Selection

```python
df["age"]          # Series
df[["name", "salary"]]  # DataFrame
```

### Row Selection

```python
df.iloc[0]     # First row by position
df.loc[0]      # By index label
```

### Filtering Rows

```python
high_earners = df[df["salary"] > 75000]
print(high_earners)
```

Logical conditions:

```python
df[(df["age"] > 25) & (df["salary"] < 90000)]
```

***

## 4. Handling Missing Values

### Detect Missing Data

```python
df.isna()
df.isna().sum()
```

### Dropping Missing Values

```python
df.dropna()
```

### Filling Missing Values

```python
df["age"] = df["age"].fillna(df["age"].mean())
```

**Common Pitfall**

*   Many pandas methods return **new DataFrames**; use `inplace=True` cautiously.

***

## 5. Grouping, Aggregating, and Transforming

### GroupBy (Split → Apply → Combine)

```python
data = {
    "department": ["IT", "HR", "IT", "HR"],
    "salary": [70000, 60000, 80000, 65000]
}

df = pd.DataFrame(data)
```

```python
df.groupby("department")["salary"].mean()
```

### Multiple Aggregations

```python
df.groupby("department").agg(
    avg_salary=("salary", "mean"),
    max_salary=("salary", "max")
)
```

### Transform (Preserve Shape)

```python
df["department_avg"] = df.groupby("department")["salary"].transform("mean")
```

***

## 6. Merging, Joining, and Reshaping

### Merging DataFrames

```python
employees = pd.DataFrame({
    "emp_id": [1, 2, 3],
    "name": ["Alice", "Bob", "Charlie"]
})

salaries = pd.DataFrame({
    "emp_id": [1, 2, 3],
    "salary": [70000, 80000, 90000]
})

merged = pd.merge(employees, salaries, on="emp_id")
print(merged)
```

### Types of Joins

*   `inner`
*   `left`
*   `right`
*   `outer`

```python
pd.merge(df1, df2, how="left")
```

***

## 7. Practical Use Cases and Best Practices

### Typical Use Cases

*   Data cleaning and preprocessing
*   Exploratory data analysis (EDA)
*   Joining multiple datasets
*   Feature engineering

### Best Practices

✅ Inspect early (`info`, `head`)  
✅ Use boolean filtering instead of loops  
✅ Prefer vectorized pandas ops  
✅ Keep data tidy (one variable per column)  
✅ Avoid chained indexing (`df[df["x"] > 3]["y"]`)

***

## How NumPy and Pandas Fit Together

*   Pandas **stores data internally as NumPy arrays**
*   You can move between them easily:

```python
arr = df["salary"].to_numpy()
df["bonus"] = arr * 0.1
```
