
# sed Practice Worksheet (using `grep_practice` dataset)

Run these from inside the `grep_practice/` directory. Each task shows the intended `sed` command and explains what it should do.

---
## ‚úÖ Easy Level

### 1) Substitute `ERROR` ‚Üí `ERR` in `log.txt` (print to stdout)
**Command:**
```bash
sed 's/ERROR/ERR/' log.txt
```
**Goal:** Only the first occurrence of `ERROR` per line is changed to `ERR`.

### 2) Global substitution: change every `INFO` ‚Üí `INFORMATION` in `log.txt`
**Command:**
```bash
sed 's/INFO/INFORMATION/g' log.txt
```
**Goal:** All `INFO` tokens become `INFORMATION`.

### 3) Case-insensitive replace `warning` ‚Üí `WARN` in `system.log`
**Command:**
```bash
sed 's/warning/WARN/Ig' system.log
```
**Goal:** Replace both `warning` and `WARNING`.

### 4) Show only lines that changed (use `-n` + `p`)
**Command:**
```bash
sed -n 's/failed/FAIL/p' report.txt
```
**Goal:** Print lines where `failed` was replaced.

## üîç Intermediate Level

### 5) Replace `port=8080` with `port=80` across `etc` recursively and show filenames changed (dry run)
**Command (preview):**
```bash
grep -rl 'port=8080' etc | xargs sed -n 's/port=8080/port=80/p'
```
**Goal:** Preview lines that would change.

### 6) In-place edit `etc/app/config.conf`: set `host=127.0.0.1`
**Command:**
```bash
sed -i 's/^host=.*/host=127.0.0.1/' etc/app/config.conf
```
**Goal:** After running, `host=127.0.0.1` should be in the file.

### 7) Delete lines containing `DEBUG` in `app.log` (print result to stdout)
**Command:**
```bash
sed '/DEBUG/d' app.log
```
**Goal:** Output without the `DEBUG` line.

### 8) Anchor: Prefix `[ERROR] ` to lines starting with `ERROR` in `server.log`
**Command:**
```bash
sed 's/^ERROR/[ERROR] &/' server.log
```
**Goal:** Only lines that begin with `ERROR` get the prefix.

### 9) Number lines for `animals.txt` using `=` and `N` formatting
**Command:**
```bash
sed = animals.txt | sed 'N;s/\n/\t/'
```
**Goal:** Show `line_number<TAB>content`.

## üí™ Advanced Level

### 10) Extract IPv4 addresses (print addresses only) from `access.log`
**Command:**
```bash
sed -n 's/^\([0-9]\{1,3\}\.[0-9]\{1,3\}\.[0-9]\{1,3\}\.[0-9]\{1,3\}\).*/\1/p' access.log
```
**Goal:** Print the first IPv4 on each line.

### 11) Replace dates `YYYY-MM-DD` with `YYYY/MM/DD` in `data.txt`
**Command:**
```bash
sed 's/\([0-9]\{4\}\)-\([0-9]\{2\}\)-\([0-9]\{2\}\)/\1\/\2\/\3/g' data.txt
```
**Goal:** Convert hyphen-separated dates to slash format.

### 12) Context addressing: In `kernel.log`, uppercase any line with `critical` (case-insensitive)
**Command:**
```bash
sed -n '/[Cc]ritical/{s/.*/\U&/p}' kernel.log
```
**Goal:** Print only those lines, uppercased.

### 13) Insert a header before the first line of `log.txt` (stdout only)
**Command:**
```bash
sed '1i\\# Log Start' log.txt
```
**Goal:** Show `# Log Start` inserted before line 1.

### 14) Append a footer after the last line of `log.txt` (stdout only)
**Command:**
```bash
sed '$a\\# Log End' log.txt
```
**Goal:** Show `# Log End` appended after the last line.

### 15) Transform letters: convert `cat` to `bat` using transliteration in `animals.txt`
**Command:**
```bash
sed 'y/cb/ba/' animals.txt
```
**Goal:** Demonstrate `y///` by turning `cat`‚Üí`bat` and `bobcat`‚Üí`boabat` (transliterates characters `c`‚Üí`b`, `b`‚Üí`a`).
