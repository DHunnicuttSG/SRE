## üß© What Is a NULL Value?

**Definition:**

> In SQL, `NULL` represents the **absence of a value** ‚Äî *not zero, not an empty string, not ‚Äúunknown‚Äù text ‚Äî but literally ‚Äúno value at all.‚Äù*

It means the database simply doesn‚Äôt know or doesn‚Äôt have a value for that column in that row.

## üí≠ Important Concept: ‚ÄúUnknown,‚Äù Not Zero or Blank

| Expression | Meaning                                          |
| ---------- | ------------------------------------------------ |

## ‚öôÔ∏è How NULL Works in SQL

### 1Ô∏è‚É£ **NULL in Comparisons**

You can‚Äôt compare NULL values using `=` or `!=`.

Example:

```sql
SELECT * FROM Students WHERE Grade = NULL;
```

‚ùå Returns nothing ‚Äî because NULL cannot be compared directly.

Instead, use **IS NULL** or **IS NOT NULL**:

```sql

```

‚úÖ Returns all students with missing grades.

---

### 2Ô∏è‚É£ **NULL in Expressions**

When a NULL is used in an expression, the result is usually **NULL**.

Example:

```sql
SELECT Grade + 5 FROM Students;
```

If `Grade` is `NULL`, the result is `NULL`.

üß† Think of it like: *Unknown + 5 = Unknown.*

---

### 3Ô∏è‚É£ **NULL in Aggregates**

Some aggregate functions **ignore** NULLs, while others don‚Äôt:

| Function          | NULL Behavior                    |
| ----------------- | -------------------------------- |
| `COUNT(*)`        | Counts all rows, including NULLs |
| `COUNT(column)`   | Ignores NULLs                    |
| `AVG(column)`     | Ignores NULLs                    |
| `SUM(column)`     | Ignores NULLs                    |
| `MIN()` / `MAX()` | Ignores NULLs                    |

Example:

```sql

```

If some grades are NULL, they‚Äôre **excluded** from the calculation.

---

### 4Ô∏è‚É£ **Replacing NULL Values**

You can replace NULLs with a default using:

* `COALESCE()` or
* `IFNULL()` (MySQL-specific)

**Example:**

```sql

  StudentName,
  COALESCE(Grade, 'N/A') AS Grade
FROM Students;
```

‚úÖ Displays ‚ÄúN/A‚Äù where Grade is NULL.

## üß© NULL and Constraints

| Constraint Type | Behavior with NULL                                                              |
| --------------- | ------------------------------------------------------------------------------- |
| `PRIMARY KEY`   | ‚ùå Cannot be NULL (every row must have a unique identifier)                      |
| `FOREIGN KEY`   | ‚úÖ Can be NULL (if no relationship applies)                                      |
| `UNIQUE`        | ‚úÖ Can be NULL, but multiple NULLs allowed (since NULL ‚â† NULL)                   |
| `NOT NULL`      | ‚ùå Disallows NULLs                                                               |
| `CHECK`         | Evaluated ‚Äî but be careful, because `NULL` in a check may yield unknown results |

Example:

```sql
CREATE TABLE Students (
  StudentID INT PRIMARY KEY,
  Name VARCHAR(100) NOT NULL,
  Grade CHAR(2) NULL
);
```

## ‚ö†Ô∏è Common Mistakes with NULL

| Mistake                               | Explanation                                    |
| ------------------------------------- | ---------------------------------------------- |
| Using `=` instead of `IS NULL`        | NULLs don‚Äôt compare equal to anything          |
| Assuming `COUNT(column)` counts NULLs | It doesn‚Äôt                                     |
| Treating NULL = 0 or ''               | They‚Äôre totally different                      |
| Forgetting about NULL in joins        | Rows might be dropped unexpectedly             |
| Using `NOT IN` with NULL values       | May yield no rows, because `NULL` is ‚Äúunknown‚Äù |

Example of a **join issue**:

```sql

FROM Students s
LEFT JOIN Grades g ON s.Grade = g.GradeCode;
```

If `s.Grade` is `NULL`, that student won‚Äôt match any row in `Grades`.
